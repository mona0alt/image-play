计费流程：成功才扣费

避免失败时扣费，支持并发安全

生成计费流程图
1. 提交生成请求
前端传入 prompt + 风格（或原图）
↓
2. 检查额度（不扣费）
检查 免费额度 > 0 或 余额 ≥ 单价
同时检查该用户是否有进行中的生成（防并发）
↓ 足够
3. 创建 generation 记录（status = generating）
写入数据库，锁定用户生成状态
↓
4. 调用模型 API 生成
通过 Higress 调用第三方模型（3~10s）
↓
5. 内容审核
同步审核结果图（~300ms）
↓
6a. 成功 + 审核通过
扣免费额度/余额 → 记流水
status = success
6b. 失败/审核不通过
不扣费
status = failed / audit_rejected
↓
7. 释放用户生成锁
用户可提交下一次生成

关键设计

  • 后扣费：生成成功且审核通过后才真正扣费，失败/审核不通过零成本。
  • 并发锁：一个用户同时只能有 1 个进行中的生成（通过数据库 status = 'generating' 或 Redis 锁），防止并发扣费冲突。
  • 超时兜底：模型 API 60s 超时，超时后标记为 failed，不扣费。
