内容审核对生成延迟的影响

两种接入方式对比

方案 X：同步审核（推荐）

文生图流程：

  1. 用户提交 prompt + 风格
  2. Go 后端扣费/检查免费额度
  3. 调用模型 API 生成（3~10s）
  4. 拿到图片 URL，调用内容审核（~300ms）
  5. 审核通过 → 返回图片给用户
  6. 审核不通过 → 返回"内容违规"，图片不入库

图生图流程：

  1. 用户上传原图
  2. 先审核原图（~200ms） → 不通过直接拒绝
  3. 调用模型生成（3~10s）
  4. 再审核结果图（~300ms）
  5. 返回结果

优点

  • 额外延迟仅 200~500ms
  • 模型生成 3~10s 是主要耗时，审核几乎可以忽略
  • 违规图片绝不会被用户看到
方案 Y：异步审核

流程：

  1. 用户提交生成请求
  2. 模型生成后立即返回给用户
  3. 后台异步审核
  4. 审核不通过 → 撤回/隐藏图片

优点

  • 零额外延迟

缺点

  • 违规图片已被用户看到（风险）
  • 需要撤回机制，代码复杂
  • 用户刚保存分享的图片突然消失，体验差

结论

推荐方案 X（同步审核）。

腾讯云数据万象图片审核平均延迟在 200~500ms，而模型生成本身需要 3~10 秒。审核耗时只占总时间的 5% 左右，用户几乎感知不到。

如果模型生成走异步队列（未来扩展），审核也可以一并放在队列里处理，不影响 API 响应。
