部署方案 V3

docker-compose 一键部署，单台轻量服务器

服务器选型

腾讯云轻量应用服务器 2核4G 6M带宽，新用户首年约 80~100 元/月。

  • 系统：Ubuntu 22.04 LTS
  • 带宽：6M（约 750KB/s，足够图片上传下载）
  • 硬盘：80GB SSD（系统 + 数据库 + Docker）

docker-compose 编排

services
  • app：Go 后端（自建镜像）
  • higress：AI 网关（官方镜像）
  • postgres：PostgreSQL 15（官方镜像）
  • nginx：反向代理 + HTTPS + 管理后台静态文件

目录结构

/root/project/image-play/
├── docker-compose.yml
├── .env                    # 环境变量（密钥、数据库密码）
├── backend/                # Go 后端代码
│   ├── Dockerfile
│   └── ...
├── admin/                  # 管理后台前端（Vue3 构建产物）
│   └── dist/
├── higress/                # Higress 配置
│   └── config/
├── nginx/                  # Nginx 配置
│   └── nginx.conf
└── data/                   # 数据持久化
    ├── postgres/           # 数据库文件
    └── logs/               # 日志

域名与证书

  • 域名：需要 ICP 备案（微信小程序要求）
  • 证书：腾讯云免费 SSL 证书，Nginx 配置 HTTPS
  • 小程序后台配置 request 合法域名

启动命令

# 克隆代码
git clone ...
cd image-play

# 配置环境变量
cp .env.example .env
# 编辑 .env：填微信 AppID、支付密钥、数据库密码、COS 密钥等

# 一键启动
docker-compose up -d

# 查看日志
docker-compose logs -f app

备份策略（简单版）

  • PostgreSQL：每日凌晨 3 点自动 dump，上传到 COS
  • COS 图片：开启版本控制，防止误删
  • 配置文件：.env 和 docker-compose.yml 提交到私有 Git 仓库
