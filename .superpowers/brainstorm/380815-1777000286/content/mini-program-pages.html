小程序页面结构

共 5 个核心页面 + 管理后台

页面清单

1

首页（文生图）

输入 prompt、选择风格、点击生成。顶部展示余额和免费额度。

底部 Tab：首页 | 图生图 | 历史 | 我的

2

图生图页

上传原图（从相册或拍照）→ 输入 prompt → 选择风格 → 生成。上传前先校验大小（≤5MB）。

3

历史记录页

瀑布流/列表展示所有生成记录。支持长按删除（软删除）、点击预览、下载到相册、分享给好友。

4

我的页

展示头像昵称、余额、充值入口、免费额度说明、隐私政策入口。

5

充值页

展示后台配置的充值套餐，选中后调起微信支付。充值成功后刷新余额。

关键交互流程

生成流程交互
  1. 用户输入 prompt → 点击"生成"
  2. 前端显示加载动画（最多 60s）
  3. 成功：展示结果图，提供"保存"和"分享"按钮
  4. 审核不通过：Toast 提示"内容违规，请修改描述"，已扣费
  5. 余额不足：弹窗"余额不足，去充值？"
  6. 被封禁：弹窗"因多次违规，10分钟内无法生成"

分享功能

  • 结果页支持 onShareAppMessage 分享给微信好友（带图片预览）
  • 不支持分享到朋友圈（小程序限制），但可引导用户下载后手动发朋友圈
  • 历史页长按图片 → "分享给好友" 或 "保存到相册"
