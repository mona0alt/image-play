安全与合规检查清单

基于微信小程序 + 图片生成 + 付费场景的风险评估

🔴 必须解决（不合规可能被封禁/罚款）

1

图片内容安全审核

风险：用户生成或上传的涉黄、涉政、暴力图片，一旦流入微信生态，小程序会被直接封禁。

建议：接入腾讯云/阿里云内容安全（图片审核）API，对所有上传的原图和生成的结果图在入库前进行审核。违规图片拦截且不展示。

2

隐私政策 + 用户协议

风险：微信小程序要求涉及用户信息和付费的小程序必须提供《隐私政策》和《用户协议》，并在首次登录时弹窗同意。

建议：准备两份法律文本，前端首次登录强制弹窗同意，不同意则无法使用。

3

ICP 备案（如涉及付费）

风险：微信小程序涉及付费功能，域名需要完成 ICP 备案，且小程序类目需要选择正确（如"工具-图片处理"）。

建议：确认服务器域名已完成 ICP 备案；小程序类目在提交审核时选择准确。

🟡 强烈建议（避免资金和数据损失）

4

支付安全加固

风险：微信支付回调可能被伪造；并发下可能出现重复充值。

建议：

  • 回调签名校验（微信 SDK 自带）
  • 订单号幂等：同一微信订单号只处理一次充值
  • 金额校验：回调金额必须和下单金额一致
  • 密钥分离：支付密钥不硬编码，走环境变量/配置文件
5

API 防刷与限流

风险：生成接口被恶意调用（刷免费额度、压垮模型 API）。

建议：

  • Higress 层配置 IP 限流 + 用户限流
  • 免费额度用完必须登录后才能生成
  • 关键接口（生成、充值）加验证码或行为校验
6

传输与存储加密

风险：HTTP 明文传输被中间人截获；数据库被拖库。

建议：

  • 全站 HTTPS（腾讯云免费证书）
  • COS 私有桶 + 临时签名 URL（防止图片直链泄露）
  • 数据库敏感字段（openid）无需额外加密，但需限制访问 IP

🟢 锦上添花（提升专业度）

7

操作审计日志

记录管理员在后台的关键操作（改价格、看用户信息），防止内部泄露。

8

用户行为风控

同设备/同 IP 频繁注册新号刷免费额度时，自动标记可疑账号。

我的建议

把 1（内容审核）、2（隐私政策）、4（支付安全）、5（API 限流） 纳入 MVP 必做项。否则要么被微信下架，要么被刷到亏损。

特别是内容审核：即使你不做主动审核，微信在代码审核或用户举报时也会要求你证明有审核机制。建议接入腾讯云数据万象的图片内容审核（价格约 1.5 元/千张，成本可忽略）。
